## Overview

This directory contains Jupyter notebooks for accessing and analysing the image data products
generated by the **Rubin LSST Prompt Processing (AP) pipeline** at USDF (US Data Facility, SLAC).
The work focuses on closing the loop between **Fink broker alerts** and the corresponding
Butler data products: difference images, templates, source catalogues, kernels, and PSFs.

The collection used throughout is:

```
LSSTCam/prompt/output-2026-02-26
```

with skymap `lsst_cells_v2` and instrument `LSSTCam`.

## Notebooks

### `2026-04-09_AccessToAp-ImagesTemplatesDIAKernels.ipynb`

**Purpose:** First exploration of the Prompt Processing Butler outputs for a single
`(visit, detector, band)` data identifier extracted from a Fink alert light curve.

**Main steps:**

1. **Butler setup** — instantiate a Butler pointing at the prompt processing collection
   and query available dataset types, filtering out bookkeeping products.
2. **Dataset schema inspection** — print the required Butler dimensions for
   `dia_source_apdb`, `dia_object_apdb`, and `dia_forced_source_apdb`.
3. **Image retrieval** — fetch `preliminary_visit_image`, `difference_image`, and
   `template_detector` for a reference data identifier tied to a known Fink alert.
4. **Firefly display** — overlay the science, DIA, and template images in three
   synchronised Firefly frames with the alert source position marked.
5. **Star footprints** — retrieve `single_visit_star_footprints` and inspect its schema.
6. **Difference kernel analysis** — fetch `difference_kernel` (a `LinearCombinationKernel`)
   and characterise it fully:
   - number and type of basis kernels,
   - spatial variation flag (`isSpatiallyVarying`),
   - kernel image evaluated at the alert position,
   - grid visualisation of all basis functions (normalised and shared-scale),
   - coefficient decomposition to diagnose PSF-matching quality.

**Key dataset types accessed:**
`preliminary_visit_image`, `difference_image`, `template_detector`,
`single_visit_star_footprints`, `difference_kernel`, `dia_source_apdb`.

### `2026-04-16_AccessToAp-ImagesAndComputeFluxes.ipynb`

**Purpose:** Close the loop between Fink broker alert fluxes and the corresponding
Butler prompt products by re-running source detection and PSF flux measurement
on the difference image, then comparing the recomputed fluxes with the pipeline values.

**Main steps:**

1. **Butler setup** — same configuration as the first notebook; additionally queries
   `new_dia_source` dimensions and lists `prompt_provenance` dataset IDs.
2. **Multi-catalogue flux comparison** — retrieve three source catalogues for the same
   data ID: `new_dia_source` (t1), `dia_source_detector` (t2), and `dia_source_apdb` (t3).
   Identify a specific `diaObjectId` from the Fink light curve and record its
   pixel coordinates `(x0, y0)` and sky position `(ra0, dec0)`.
3. **Image retrieval** — fetch science image, background, difference image
   (+ masked image), and template; display all five frames in Firefly.
4. **Photometric calibration** — access the `PhotoCalib` object from the difference image
   to recover the zero-point and convert between instrumental and physical fluxes.
5. **DIA image re-processing** — manually re-run the LSST stack detection and
   measurement pipeline on the difference image:
   - Build a minimal `SourceTable` schema and configure `SourceDetectionTask`
     with bilateral polarity detection (`thresholdPolarity = "both"`).
   - Run `SingleFrameMeasurementTask` to compute `base_PsfFlux_instFlux`.
   - Check the sign distribution of detected fluxes (positive vs negative DIA sources).
   - Convert instrumental fluxes to magnitudes via `photoCalib`.
   - Plot flux and magnitude histograms for positive and negative sources.
6. **Footprint inspection** — retrieve and display source footprints overlaid on the
   difference image (red rectangles = positive flux, blue = negative flux).
7. **WCS cross-check** — convert pixel centroids to sky coordinates and compute
   angular separations to the reference Fink source.
8. **PSF kernel visualisation** — extract the spatially-varying PSF from the difference
   image and visualise the kernel image at each source position.

**Key dataset types accessed:**
`preliminary_visit_image`, `preliminary_visit_image_background`, `difference_image`,
`template_detector`, `new_dia_source`, `dia_source_detector`, `dia_source_apdb`,
`single_visit_star_footprints`, `prompt_provenance`.

**Key LSST tasks used:**
`SourceDetectionTask`, `SingleFrameMeasurementTask`, `PhotoCalib`.

## Common Infrastructure

Both notebooks share the following utility functions defined in the imports section:

- **`get_uris_from_butler(butler, datasetType, collections, **kwargs)`** — returns a list
  of file URIs for a given dataset type, useful for direct filesystem access.
- **`get_refs_from_butler(butler, datasetType, collections, return_uris, **kwargs)`** — 
  safely converts numpy integer types before passing them to Butler registry queries,
  avoiding common type-mismatch errors.

## Scientific Context

These notebooks are part of a broader effort to **validate Fink broker alert fluxes**
(`psfFlux`, `scienceFlux`, `templateFlux`) against the USDF prompt processing outputs,
with the goal of understanding flux variability in Gaia DR3 calibration stars.
The work connects to the `04_calib/` photometric calibration pipeline developed at IJCLab.

Key reference for the difference kernel formalism:
[DMTN-021](https://dmtn-021.lsst.io/v/v1/index.html)

Key reference for source detection and measurement:
[DP1 tutorial 105.2](https://dp1.lsst.io/tutorials/notebook/105/notebook-105-2.html)
